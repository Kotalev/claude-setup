# E2E and Out-of-Scope Test Exclusions

The `refresh-tests` skill only audits **unit and integration tests**. End-to-end tests, load tests, and performance benchmarks are out of scope because:

1. They require running services, browsers, or real backends — too slow for an audit cycle
2. They have different failure semantics (flaky, environment-dependent)
3. They have dedicated tooling that the user manages separately

## Path Patterns (Excluded)

The `inventory-tests.sh` script excludes any test whose path contains any of these case-insensitive substrings:

| Pattern | Tool / purpose |
|---------|----------------|
| `/e2e/` | Generic end-to-end test directory |
| `/__e2e__/` | Same, conventional Jest-style |
| `/e2e-tests/` | Variation |
| `/integration-e2e/` | Variation |
| `/playwright/` | Playwright project layout |
| `/cypress/` | Cypress project layout |
| `/selenium/` | Selenium WebDriver |
| `/webdriver/` | WebdriverIO |
| `/load-tests/`, `/load_tests/` | k6, Artillery, Locust, JMeter |
| `/perf-tests/`, `/perf_tests/` | Performance benchmarks |
| `/stress-tests/` | Stress / soak tests |
| `/smoke-tests/` | Often E2E-style smoke runs |
| `/.storybook/` | Storybook stories (not tests) |

## File Name Patterns (Excluded)

| Pattern | Tool |
|---------|------|
| `*.e2e.*` | Generic |
| `*.e2e-spec.*` | Angular CLI / Protractor |
| `*.spec.e2e.*` | Variation |
| `*.cy.{js,ts,jsx,tsx}` | Cypress component/e2e tests |
| `*.pw.{js,ts}` | Playwright (when not in `/playwright/` dir) |

## Special Cases

### Storybook stories (`*.stories.*`)
Stories visualise components but aren't tests. Excluded by `/.storybook/` path; loose `*.stories.tsx` files in component dirs are NOT auto-excluded — they're not picked up by `*.test.*` / `*.spec.*` patterns anyway.

### Snapshot files (`__snapshots__/`)
Auto-generated by Jest/Vitest. Not test files themselves. Skip during inventory; surface separately as "snapshot needs review" if outdated.

### Fixtures (`fixtures/`, `__fixtures__/`)
Test data files — not tests. Not matched by test patterns by default.

### Manual test scripts (`scripts/test-*.sh`, `manual-tests/`)
Out of scope; user runs these manually.

## Why these specific patterns?

We chose substring + glob matching (rather than runner-config introspection) because:

1. Many projects don't formally separate E2E config — they just put files in `e2e/` directory
2. Cypress/Playwright naming conventions are more reliable than their config files
3. Substring matching is fast and language-agnostic

The trade-off: a unit test misnamed `something.e2e.js` will be excluded. This is acceptable — a unit test in a clearly E2E-named file is itself a problem.

## When to override

If the audit reports too few tests, check the stderr output of `inventory-tests.sh` — it lists every excluded file as `excluded:e2e <path>`. If something legitimate was excluded, ask the user before adjusting; do not silently add false-positives back.

## Producer-specific notes

### Angular projects
Angular CLI generates `*.spec.ts` for unit tests AND `*.e2e-spec.ts` for E2E. Default config separates them; we match the same convention.

### Next.js / React
Storybook stories often live next to components (`Foo.stories.tsx`). Not matched by `*.test.*` / `*.spec.*` so no special exclusion needed.

### Mono-repo with shared E2E suite
A common layout: `apps/web/` (unit tests inside), `apps/api/` (unit tests inside), `e2e/` (Cypress at repo root). The `e2e/` directory is excluded by path; per-app unit tests are kept.
